1. Project Overview
Tazizi requires a lightweight pre-order web application that allows customers to browse and place food orders ahead of time. The app will be primarily accessed by customers scanning a QR code placed on a fridge at a physical Tazizi location - each location will have a unique QR code. Once scanned, the app remembers the customer's location via browser cookies, providing a seamless, location-aware ordering experience. 
The developer is also responsible for extending the existing Tazizi admin panel to support day-specific menu management — allowing the admin to configure which menu items are available on specific days of the week.


2. Scope of Work
2.1  Pre-Order Web App (Front-End)
* Build a responsive, mobile-first web app in React / Next.js
* Appealing design per Tazizi logo/colors
* Display available menu items based on the current day's / week’s schedule set through the admin panel
* Display at top: 10% discount when pre-ordering for entire week (apply 10% discount when selecting a meal for each day of the week) - we must be able to remove then from the admin panel or change the discount[a]
* Allow users to browse items, select quantities, and submit a pre-order
* Read location from cookie on page load; prompt if none is set
* Store and retrieve the scanned location identifier in browser cookies
* Show confirmation screen upon successful order submission
* Basic loading and error states throughout the UI
* Payment processing or integration with any payment gateway (Stripe) - MUST BE SECURE. IDEALLY WE ARE REMOVE FROM ANY CC INPUTS, ONLY DIRECTLY THROUGH STRIPE
* User to receive email receipt
* Required fields during checkout: Phone/Whatsapp, email, Name (as displayed on Meal)




2.2  QR Code & Location Logic
* Each physical fridge location has a unique QR code containing a location ID in the URL (e.g. /order?location=marina-01)
* On first scan, the app reads the location ID from the URL and stores it in a cookie
* On return visits (without QR), the app reads the cookie and auto-selects the location
* Location name is displayed on the order page so the customer can confirm correctness


2.3  Admin Panel Integration
* Extend the existing admin panel with a day-by-day-specific menu scheduler
* Admin can assign menu items to one or more days of the week (Mon–Sun)
* Admin can toggle items active/inactive per day without deleting them
* Admin can set discount x% for Y meals per week pre-order. Discount heading on UX should only be shown when discount is active
* Pre-order app reads the current day and only shows items scheduled for that day
* Changes in the admin panel reflect in the live pre-order app in real time (or near real time)
* Preorders must be listed in the admin panel to convert to QR codes[b]

6. Assumptions & Requirements
* The developer will be given read/write access to the existing admin panel codebase and test database
* Tazizi team will provide location IDs and QR code format/structure
* Feedback and review from the Tazizi team will be provided within 24 hours of each milestone
* The developer is responsible for their own local development environment setup
* Hosting and deployment are managed by the Tazizi team; developer handles code only


7. Acceptance Criteria
This project will be considered complete when all of the following conditions are met:


* Scanning a location QR code lands the user on the pre-order page with the correct location pre-selected
* Returning to the app without scanning retains the last saved location via cookie
* Only menu items assigned to the current day are displayed to the user
* A user can successfully submit a pre-order and see a confirmation screen
* Admin can log into the admin panel and assign menu items to specific days
* All code is pushed to the repository with a README covering setup and deployment.
